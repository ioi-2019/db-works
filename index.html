Table participations
(This table stores the access of users to the contests.)
Idx Field Name Data Type Description
* id int A unique ID of each participation
ip varchar If access is login based, then the IP or mask is stored here
starting_time varchar the start time of the access
* delay_time varchar
* extra_time varchar an extension time for the particular user
password varchar a password (plan or encrypted) for accessing a contests
* hidden varchar if '1', then the scores of the user will not be shown in Ranking
* unrestricted varchar
* contest_id int Unique ID of the contest
* user_id int Unique ID of the user
team_id int Unique ID of the user's team
Indexes
ix_participations_contest_id ON contest_id
ix_participations_user_id ON user_id

Table teams
(Stores the list of teams, like 'AZE', 'FRA' and 'USA')
Idx Field Name Data Type Description
* id int The unique ID of the team
* code varchar A short description of the team. In IOI, it's a country code in ISO 3166-1 alpha-3 code format
* name varchar Full description of the team. In IOI, the name of the country.

Table users
(This table stores the list of all entered users. Adding a user to CMS does not guarantee the access to the contests. To grant an access to the contests, user shall exists in a participation table. )
Idx Field Name Data Type Description
* id int A unique ID of the user
* first_name varchar First name
* last_name varchar Last name
* username varchar Login name of the user. In IOI, it's usually associated with the country code. Example: 'AZE1', 'ITA3' and 'SWE2'.
Caution: avoid using "-" sign in logins, like 'SGP-4'.
* password varchar A password of the user is stored here. It can be plain or encrypted text
email varchar Email of the user
timezone varchar Timezone of the user
* preferred_languages varchar
